## One sentence in, a workflow out

```bash
sweny workflow create "scan our repo for security issues, check dependencies \
  for vulnerabilities, and create Linear tickets for anything critical"
```

SWEny generates a full DAG — with nodes, conditional routing, structured output schemas, and the right skills wired up at each step. Refine it with natural language, run it immediately, or save it for later.

```
  ┌────────────────────────────────────────┐
  │ ○ Scan Commits for Exposed Secrets     │
  └───────────────────┬────────────────────┘
                      │
         ┌────────────┴────────────────────────────┐
         │                                         │
  ┌──────▼─────────────────────┐  ┌────────────────▼───────────────────┐
  │ ○ Review PRs for Security  │  │ ○ Scan Dependencies for Vulns     │
  └──────┬─────────────────────┘  └────────────────┬───────────────────┘
         │
  ┌──────▼─────────────────────────┐
  │ ○ Compile Security Report      │
  └──────┬─────────────────────────┘
         │
  ┌──────▼─────────────────────────┐
  │ ○ Create Linear Tickets        │
  └────────────────────────────────┘
```

This isn't a template. It was generated from that one sentence, with detailed instructions, output schemas, and conditional edges at every node.

## Build, refine, run

1. **`sweny workflow create "..."`** — describe your task in plain English, get a complete workflow
2. **`sweny workflow edit my-workflow.yml "add a quality gate"`** — modify workflows with natural language
3. **`sweny workflow run my-workflow.yml`** — execute it with live progress, streaming events, and full observability

No YAML to write by hand. No boilerplate to configure. Describe what you want, refine until it's right, and run it.

## Not just code tasks

SWEny works for anything you can describe as steps with clear inputs and outputs:

- **Content pipelines** — generate blog posts, run them through LLM quality judges, publish passing content
- **Competitive analysis** — research competitors, gather pricing/features, synthesize a comparison matrix, produce an executive brief
- **Product launch planning** — research recent launches, draft copy with quality gates, create a Linear checklist
- **Security audits** — scan commits, review PRs, check dependencies, file tickets for findings

These are real workflows generated from a single `sweny workflow create` command.

## How you use it

**[CLI](/cli/)** — build and run workflows from your terminal. This is the primary way to create workflows and get things done. Supports live progress, dry-run, streaming output, and all configured providers.

**[GitHub Action](/action/)** — deploy workflows to CI. Run on cron schedules, dispatch events, or webhook triggers. The built-in triage workflow monitors production errors and opens fix PRs automatically.

**[Studio](/studio/)** — visual DAG editor and live execution monitor. Watch workflows run node-by-node in real time, or build them visually by dragging nodes and connecting edges.

**[SWEny Cloud](https://app.sweny.ai)** — the managed platform for teams. Dashboard, shared credentials, scheduling, and cross-repo analytics. [See pricing](https://app.sweny.ai/#/pricing).

## Quick install

```bash
npm install -g @sweny-ai/core
sweny workflow create "your task description here"
sweny workflow run .sweny/workflows/your-workflow.yml
```
