# Task 24 — Studio: Export workflow as GitHub Actions YAML

## Goal

Users design a workflow in Studio and then want to deploy it as a scheduled
GitHub Actions job. Today they have to manually write the `.github/workflows/`
YAML. This task adds a "↓ GitHub Actions" export button that generates a
ready-to-commit workflow file.

This is a **conversion + retention** feature: shortens the path from
"I designed a workflow" to "it's running in CI" from hours to seconds.

## Context

- **`packages/studio/src/lib/`** — Add `export-github-actions.ts` here
- **`packages/studio/src/components/Toolbar.tsx`** — Add export button alongside the existing YAML/JSON/TypeScript buttons
- The generated file should be a complete GitHub Actions workflow YAML that:
  1. Installs Node.js and the `@sweny-ai/cli` package
  2. Runs `sweny triage` with environment variables populated from `${{ secrets.* }}`
  3. Has a sane default schedule (daily at 06:00 UTC) and manual dispatch trigger

The generated YAML should look like:

```yaml
# Generated by SWEny Studio — https://studio.sweny.ai
# Workflow: {name} ({id})
name: SWEny — {name}

on:
  schedule:
    - cron: "0 6 * * *"   # Daily at 06:00 UTC — adjust as needed
  workflow_dispatch:        # Manual trigger

jobs:
  sweny:
    runs-on: ubuntu-latest
    permissions:
      contents: write       # For creating branches and committing code
      pull-requests: write  # For creating PRs
      issues: write         # For creating/commenting on issues

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: "20"

      - name: Install SWEny CLI
        run: npm install -g @sweny-ai/cli

      - name: Run SWEny triage
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          # Add more secrets based on your providers:
          # DD_API_KEY: ${{ secrets.DD_API_KEY }}
          # DD_APP_KEY: ${{ secrets.DD_APP_KEY }}
          # LINEAR_API_KEY: ${{ secrets.LINEAR_API_KEY }}
        run: sweny triage
```

**Implementation notes:**
- Detect which step types are in the workflow (look at `step.type` fields)
- Use `uses` field on steps to determine which providers are needed
- Include the relevant `# Add more secrets` comments for each provider detected in the workflow
- If the workflow contains `sweny/investigate` or `sweny/implement-fix`, include `ANTHROPIC_API_KEY`
- If any step uses `"observability"`, add a comment block for DD_API_KEY/DD_APP_KEY
- If any step uses `"issueTracker"` and workflow name hints linear, add LINEAR_API_KEY comment
- The file should use `yaml` string building (no yaml library needed — template strings are fine for this well-defined output)

### Toolbar.tsx changes

Add a new export button next to the existing ones:

```tsx
<button onClick={handleExportGitHubActions} className="px-3 py-1 rounded bg-gray-700 hover:bg-gray-600 text-xs">
  ↓ GitHub Actions
</button>
```

Add the handler in Toolbar.tsx (same pattern as `handleExportYaml`):

```typescript
function handleExportGitHubActions() {
  const { definition: def } = useEditorStore.getState();
  const content = exportAsGitHubActions(def);
  const blob = new Blob([content], { type: "text/yaml" });
  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = `sweny-${def.id}.yml`;
  a.click();
  URL.revokeObjectURL(url);
}
```
