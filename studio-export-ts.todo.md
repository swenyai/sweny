# Task: Studio — Export Recipe as TypeScript

## Goal
Add an "Export TS" button to the Toolbar that generates a complete, ready-to-use
TypeScript file from the current recipe definition and downloads it.

The generated file should:
1. Import the correct types from `@sweny-ai/engine`
2. Export the `RecipeDefinition` as a const (using the same pattern as `triage/definition.ts`)
3. Export stub implementations for every state — typed, with comments showing which outcomes each state routes to
4. Export the final `createRecipe(definition, implementations)` call

## Reference: what a real recipe looks like

Look at these files to understand the exact structure to generate:
- `packages/engine/src/recipes/triage/definition.ts` — the pure definition export pattern
- `packages/engine/src/recipes/triage/index.ts` — how createRecipe is called with implementations
- `packages/engine/src/types.ts` — WorkflowContext, StepResult, StateImplementations types

## Generated output example

Given a recipe `{ id: "my-flow", name: "My Flow", states: { "step-a": { phase: "learn", on: { success: "step-b", failed: "end" } }, "step-b": { phase: "act", next: "end" } }, initial: "step-a" }`, generate:

```typescript
/**
 * My Flow — generated by sweny studio
 * Recipe ID: my-flow | Version: 1.0.0
 */
import { createRecipe } from "@sweny-ai/engine";
import type { RecipeDefinition, WorkflowContext, StepResult } from "@sweny-ai/engine";

export const myFlowDefinition: RecipeDefinition = {
  id: "my-flow",
  version: "1.0.0",
  name: "My Flow",
  initial: "step-a",
  states: {
    "step-a": {
      phase: "learn",
      on: { success: "step-b", failed: "end" },
    },
    "step-b": {
      phase: "act",
      next: "end",
    },
  },
};

// ─── Implementations ───────────────────────────────────────────────────────────
// Return { status: "success" | "skipped" | "failed", data?: { outcome?: string } }
// Use data.outcome to drive named on: transitions.

async function stepA(ctx: WorkflowContext<unknown>): Promise<StepResult> {
  // Outcomes: success → step-b | failed → end
  void ctx;
  return { status: "success" };
}

async function stepB(ctx: WorkflowContext<unknown>): Promise<StepResult> {
  // Outcomes: (default) → end
  void ctx;
  return { status: "success" };
}

// ─── Recipe ────────────────────────────────────────────────────────────────────

export const myFlowRecipe = createRecipe(myFlowDefinition, {
  "step-a": stepA,
  "step-b": stepB,
});
```

## Implementation steps

### 1. Create `packages/studio/src/lib/export-typescript.ts`

Pure function — no React, no store imports:

```typescript
import type { RecipeDefinition } from "@sweny-ai/engine";

/**
 * Generate a TypeScript recipe file from a RecipeDefinition.
 * Output is ready to drop into a Node.js project that has @sweny-ai/engine installed.
 */
export function exportAsTypescript(definition: RecipeDefinition): string { ... }
```

Logic:
- Convert recipe `id` to a camelCase identifier for variable names (e.g. `"my-flow"` → `myFlow`)
- Convert state ids to camelCase function names (e.g. `"verify-access"` → `verifyAccess`)
- Serialize the definition object as a TypeScript `const` (NOT JSON.stringify — use proper indented TS object literal notation matching how our definition.ts files look)
- For each state, generate an async function stub
  - Comment with outcomes: for each `on` key → target, and `next` → target
  - Skip `"end"` in comments (it's a built-in terminal)
  - Use `void ctx;` to suppress unused variable warnings
- Generate the `createRecipe` call at the bottom

Helper needed: `toCamelCase(str: string): string` — handles kebab-case, snake_case, spaces.

**DO NOT use JSON.stringify for the definition** — the output must be TypeScript syntax (no quotes around property keys that are valid identifiers, proper indentation).

### 2. Add "Export TS" button to `Toolbar.tsx`

Find the "Export JSON" button (already exists) and add an "Export TS" button next to it:

```tsx
function handleExportTs() {
  const { definition } = useEditorStore.getState();
  const content = exportAsTypescript(definition);
  const blob = new Blob([content], { type: "text/plain" });
  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = `${definition.id}.ts`;
  a.click();
  URL.revokeObjectURL(url);
}
```

### 3. Test locally

Run `npm run typecheck` in packages/studio to verify no type errors.
Run `npm run build` in packages/studio to verify clean build.

Optionally, add a unit test in `packages/studio/src/lib/export-typescript.test.ts`:
- Given the triage definition, the output contains `createRecipe`
- The output contains a function stub for each state
- The output does not contain a stub for `"end"`
