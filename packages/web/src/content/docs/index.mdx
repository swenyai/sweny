## How it works

SWEny Triage runs on a schedule (or manual trigger) and executes three phases:

1. **Investigate** — Query your observability platform, analyze errors, check your issue tracker for known issues, and rank by impact
2. **Implement** — Create or update a ticket, create a branch, have Claude write the fix, and open a PR ready for review
3. **Notify** — Post a summary to your GitHub Actions run with the full investigation log

Before acting, SWEny searches your issue tracker and open PRs for existing matches. If the error is already tracked, SWEny adds a "+1 occurrence" comment with the latest count instead of filing a duplicate.

## Install the provider library

```bash
npm install @sweny/providers
```

```typescript
import { datadog } from "@sweny/providers/observability";
import { linear } from "@sweny/providers/issue-tracking";
import { github } from "@sweny/providers/source-control";
```

Every provider follows the same pattern: import the factory function, pass your config, and use the returned object. [Build your own](/getting-started/providers/) by implementing the TypeScript interface.
