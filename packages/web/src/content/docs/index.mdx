## Create it

```bash
sweny workflow create "scan our repo for security issues, check dependencies \
  for vulnerabilities, and create Linear tickets for anything critical"
```

SWEny generates a full workflow — nodes, conditional routing, structured output schemas, and the right tools wired up at each step. Refine it with natural language until it's right.

## Run it

```bash
sweny workflow run .sweny/workflows/security_audit.yml --stream
```

Watch it execute node-by-node with live progress. Pass `--stream` to pipe NDJSON events into [Studio](/studio/) and see the DAG light up in real time.

```
  ✓ [1/5] Scan Commits for Exposed Secrets     12s
    ↳ Searched 847 commits, 0 secrets found
  ✓ [2/5] Review PRs for Security Changes      28s
    ↳ Reviewed 3 open PRs, flagged auth middleware change
  ⠹ [3/5] Scan Dependencies for Vulns          6s
    ↳ Calling github.get_file for package-lock.json
```

That's it. Describe a task, run it, see results. No YAML to write by hand, no boilerplate, no config files.

## Not just code

SWEny works for anything you can describe as steps with clear inputs and outputs:

- **Content pipelines** — generate blog posts, run them through LLM quality judges, publish passing content
- **Competitive analysis** — research competitors, gather pricing/features, synthesize a comparison, produce an executive brief
- **Security audits** — scan commits, review PRs, check dependencies, file tickets for findings
- **Product launch planning** — research launches, draft copy with quality gates, create a Linear checklist

These are real workflows generated from a single `sweny workflow create` command.

## Refine with natural language

Don't like the result? Edit it the same way you created it:

```bash
sweny workflow edit security_audit.yml "add a quality gate that rejects \
  vague findings, and loop back to re-scan if rejected"
```

SWEny adds the node, wires up the conditional edge, and shows you the updated DAG. No YAML surgery.

## One tool, many surfaces

**[CLI](/cli/)** — build and run workflows from your terminal. The primary interface.

**[GitHub Action](/action/)** — deploy workflows to CI. The built-in triage workflow monitors production errors and opens fix PRs on a schedule.

**[Studio](/studio/)** — visual DAG editor and live execution monitor. Watch workflows run node-by-node.

**[SWEny Cloud](https://app.sweny.ai)** — dashboard, shared credentials, scheduling, and analytics for teams.

## Get started

```bash
npm install -g @sweny-ai/core
sweny workflow create "your task description here"
sweny workflow run .sweny/workflows/your-workflow.yml
```
