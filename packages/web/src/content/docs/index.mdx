
```bash
sweny workflow create "scan our repo for security issues, check dependencies \
  for vulnerabilities, and create Linear tickets for anything critical"
```

## Not just code

SWEny works for anything you can describe as steps with clear inputs and outputs. These are real workflows generated from a single command:

- **Security audits** — scan commits, review PRs, check dependencies, file tickets for findings
- **Content pipelines** — generate blog posts, run them through LLM quality judges, publish passing content
- **Competitive analysis** — research competitors, gather pricing/features, synthesize a comparison, produce an executive brief
- **Production triage** — monitor errors, investigate root causes, create issues, open fix PRs

Don't like the result? Edit it the same way you created it:

```bash
sweny workflow edit security_audit.yml "add a quality gate that rejects \
  vague findings, and loop back to re-scan if rejected"
```

## One tool, many surfaces

|                                         |                                                                                  |
| --------------------------------------- | -------------------------------------------------------------------------------- |
| **[CLI](/cli/)**                        | Build and run workflows from your terminal. The primary interface.               |
| **[GitHub Action](/action/)**           | Deploy workflows to CI. Monitors production errors, opens fix PRs on a schedule. |
| **[Studio](/studio/)**                  | Visual DAG editor and live execution monitor. Watch workflows run node-by-node.  |
| **[SWEny Cloud](https://app.sweny.ai)** | Dashboard, shared credentials, scheduling, and analytics for teams.              |
