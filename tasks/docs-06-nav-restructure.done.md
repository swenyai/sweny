# docs-06: Restructure sidebar navigation

## Goal
Flatten and improve the docs sidebar in `packages/web/astro.config.mjs` so that new users see a clear path to value, advanced/internal sections are clearly labeled as such, and the Live Recipe Explorer is discoverable.

## Context
Current sidebar structure:
```
Getting Started
  Introduction
  Concepts
  Engine & Recipes
  End-to-End Walkthrough
  Provider Architecture
  Deploying the Agent
  Troubleshooting

Recipes
  SWEny Triage
  Inputs
  Outputs
  Examples
  Service Map

CLI
  Quick Start
  Inputs
  Examples

Agent Reference
  Plugin System
  Built-in Plugins
  Model Architecture  ← being deleted in docs-04
  Configuration

Studio
  Overview
  Recipe Authoring
  Live Recipe Explorer

Provider Reference
  (autogenerated)
```

Problems:
1. **Getting Started** has 7 items — too long. "Deploying the Agent" is about the self-hosted Slack bot, not the Action. It doesn't belong in the primary getting started flow.
2. **Recipes section** mixes recipe docs (`SWEny Triage`) with Action inputs/outputs/examples — confusing. Action docs are not "recipes".
3. **Live Recipe Explorer** is buried at the bottom of Studio — it should be higher up, it's a great first impression.
4. **Agent Reference** → should be renamed to **Advanced** and signal it's for self-hosted deployments.
5. **"Engine & Recipes"** and **"Provider Architecture"** in Getting Started add noise for typical users.

## File to edit
`packages/web/astro.config.mjs`

## New sidebar structure

```
Getting Started
  Introduction           (slug: getting-started)
  Concepts               (slug: getting-started/concepts)
  Walkthrough            (slug: getting-started/walkthrough)

Triage Recipe
```

## Notes on specific changes

### "Triage Recipe" section label
Rename from "Recipes" to "Triage Recipe" — it's more specific and honest about what's there. When more recipes are added, this can become "Recipes" again with sub-grouping.

### Remove "Engine & Recipes" and "Provider Architecture" from Getting Started
Move them to the new "Advanced" section. Typical users don't need them. They still exist and are accessible, just not in the primary path.

### Remove "Deploying the Agent" from Getting Started
Move to Advanced. This is for the self-hosted Slack bot, not the GitHub Action.

### Studio section: move Explorer to top
The Live Recipe Explorer is a great visual hook. Moving it to the top of the Studio section makes it more discoverable.

### Advanced section label
Name it "Advanced" not "Agent Reference" — it covers more than the agent (engine, providers SDK), and "Agent Reference" sounds like AI agent docs to new users.

### collapsed: true on Advanced (optional)
Consider adding `collapsed: true` to the Advanced group so it starts collapsed in the sidebar. Starlight supports this:
```js
{ label: "Advanced", collapsed: true, items: [...] }
```

`packages/web` is private. No `.changeset/` file needed.
