## What it does

SWEny runs as a scheduled GitHub Action. Every time it runs, it:

1. **Queries your logs** — pulls errors from Datadog, Sentry, or CloudWatch for the last 24 hours
2. **Checks for duplicates** — searches your issue tracker (Linear or GitHub Issues) and open PRs to avoid double-filing
3. **Investigates** — the agent reads the error logs, traces root causes through your codebase, and ranks by impact
4. **Files a ticket** — creates a detailed issue with root cause analysis, affected files, and suggested approach
5. **Writes a fix** — creates a branch, implements the fix, and opens a PR ready for human review
6. **Reports back** — posts a summary to your GitHub Actions run (or Slack, Teams, Discord)

No new infrastructure. No servers to run. Just a workflow file in your repo.

## Two deployment modes

**GitHub Action** — runs on a cron schedule in your CI. No servers, no infrastructure. This is the primary deployment.

**Interactive Agent** — a bot your team can chat with. Ask it to investigate errors, query logs, or write fixes interactively. Currently supports Slack, with more channels planned. Deploy as a container or run locally as a CLI.

## Quick install

```yaml
# .github/workflows/sweny-triage.yml
- uses: swenyai/sweny@v0.1
  with:
    claude-oauth-token: ${{ secrets.CLAUDE_OAUTH_TOKEN }}
    dd-api-key: ${{ secrets.DD_API_KEY }}
    dd-app-key: ${{ secrets.DD_APP_KEY }}
    linear-api-key: ${{ secrets.LINEAR_API_KEY }}
    linear-team-id: ${{ vars.LINEAR_TEAM_ID }}
```
